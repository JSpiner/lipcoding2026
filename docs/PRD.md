## 1. 한 줄 요약

키보드 없이 **말로 설명하면 다이어그램이 실시간으로 그려지고, 말로 수정**되는 웹 앱.
개발자는 시스템 아키텍처·시퀀스를, 기획자는 업무 프로세스 플로우를 회의 중에 즉석으로 시각화한다.

- **제품명:** 말그림 (MalGrim) — *SpeakDraw*
- **태그라인:** "말하면 그려집니다"
- **핵심 동작:** 음성 명령 → Copilot SDK 에이전트가 다이어그램 IR(구조화 JSON)을 도구 호출로 편집 → Mermaid로 렌더 → 음성으로 계속 수정/타입 전환/내보내기

## 2. 문제 정의 & 타깃 사용자

다이어그램은 소통 효율을 극적으로 높이지만, **그리는 비용이 너무 크다.** 마우스로 박스를 끌고 화살표를 잇는 동안 회의의 흐름이 끊긴다. 결국 "나중에 그릴게요"로 미뤄지고, 문서는 텍스트로만 남는다.

| 타깃 | 페인 포인트 | 말그림이 주는 가치 |
|---|---|---|
| **개발자/아키텍트** | 설계 논의 중 아키텍처/시퀀스를 그리기 번거로워 말로만 끝남 | 말하는 즉시 시퀀스/플로우가 떠서 논의를 시각으로 고정 |
| **기획자/PM** | 업무 프로세스 플로우 작성에 도구 학습·드래그 노동 필요 | 자연어로 프로세스를 말하면 플로우차트 완성, 문서에 바로 첨부 |

**핵심 인사이트:** 다이어그램 작성은 본질적으로 "구조를 말로 설명할 수 있는" 작업이다. 손이 아니라 **말**이 더 빠른 입력 수단인 거의 유일한 시각화 영역 → 보이스 + AI 에이전트에 최적.

## 4. 핵심 기능 (MVP 스코프)

1. **음성으로 다이어그램 생성** — "주문 처리 흐름을 플로우차트로 그려줘" → 에이전트가 구조를 만들고 렌더.
2. **음성으로 증분 편집** — 노드 추가/삭제, 연결, 라벨 변경, 방향 변경. "그 노드", "결제 단계" 같은 **자연어 참조를 IR 요소로 해석**.
3. **다이어그램 타입 지원** — 플로우차트(`flowchart`), 시퀀스 다이어그램(`sequenceDiagram`).
4. **타입 전환 (데모 하이라이트)** — "이걸 시퀀스로 바꿔줘" → 현재 다이어그램의 의미를 대상 타입으로 재구성.
5. **스트리밍 피드백** — 에이전트의 도구 호출과 다이어그램 변화가 실시간으로 화면에 반영.
6. **내보내기** — Mermaid 텍스트 / PNG(SVG) 다운로드 → 문서·노션·PR에 바로 첨부.
7. **텍스트 폴백** — 마이크 불가 환경/시끄러운 데모장 대비, 동일 명령을 텍스트로 입력 가능.

### Non-Goals (YAGNI — 4시간 사수를 위한 의도적 제외)

- 사용자 인증/계정, 다이어그램 영속 저장(DB) — **단일 세션 인메모리**로 충분.
- 실시간 협업/멀티유저.
- 플로우차트·시퀀스 외 다이어그램 타입(ERD/클래스/간트 등) — 시간 남으면 stretch.
- 모바일 최적화, 다국어 UI, 테마 커스터마이즈.

## 6. 아키텍처

```
[브라우저 / Next.js]
  ├─ 마이크 입력 ──▶ Azure Speech-to-Text ──▶ 텍스트 명령
  │                      └─ 음성 정제/교정 레이어(오타·오인식 보정)
  │                                   (텍스트 폴백: 입력창)
  ├─ Mermaid.js 렌더러  ◀── 현재 IR을 직렬화한 Mermaid 텍스트
  └─ 스트리밍 UI(도구 호출 로그 + 다이어그램)
        │  명령 + 현재 IR(컨텍스트)
        ▼
[백엔드 / Next.js API Route (Node, TS)]
  └─ Copilot SDK 에이전트 루프
       ├─ 모델 계층: Azure OpenAI (gpt-4o 계열 배포)
       ├─ 도구(tools): IR을 변경하는 함수들
       ├─ 컨텍스트: 현재 IR + 최근 대화
       └─ 스트리밍: 도구 호출/결과를 토큰 단위로 전송
        │  변경된 IR
        ▼
[IR 스토어 (인메모리, 세션 단위)]  ──▶  [직렬화기 IR→Mermaid]  ──▶ 브라우저 렌더
```

- **언어/스택 일원화:** TypeScript 단일 언어(Next.js 풀스택) → 보이스 코딩 속도 극대화, Copilot 보조 품질↑.
- **결정적 렌더 파이프라인:** 에이전트는 **IR(JSON)만** 변경. IR→Mermaid 직렬화는 순수 함수라 출력이 항상 유효 → 라이브 데모 안정성.

### 6.1 모델 계층 — Azure 정합성 확보 (사전 확인 필수)

- **1순위:** Copilot SDK가 Azure OpenAI 엔드포인트를 모델 백엔드로 지정할 수 있으면 → 그대로 구성(심사 3번 만점 경로).
- **차선(미지원 시):** 에이전트 오케스트레이션/도구 호출은 Copilot SDK가 담당하고, **의미 있는 AI 작업을 Azure OpenAI로 분리**해 18%를 확보한다. 예:
  - 자연어 → IR 의도 보강(모호한 요청 구조화),
  - "누락된 단계 제안" / "이 다이어그램 한 줄 설명 생성",
  - 타입 전환 시 의미 재구성.
- 둘 중 어떤 경로든 **모델 추론이 Azure OpenAI 위에서 실제로 일어나도록** 한다(단순 끼워넣기 금지 — 심사 기준 명시 감점 항목).

## 7. Copilot SDK 에이전트 설계 (심사 25% 핵심)

### 7.1 도구(Tools)

> 모든 도구는 **id 또는 자연어 라벨**을 받아 IR 요소를 식별한다. 라벨은 리졸버가 퍼지 매칭("그 노드", "결제 단계")으로 해석 → 음성 UX 안정.

| 도구 | 시그니처(개념) | 설명 |
|---|---|---|
| `create_diagram` | `(type: "flowchart"\|"sequence", title?)` | 새 다이어그램 시작 |
| `add_node` | `(label, shape?) → id` | 플로우차트 노드 추가 |
| `add_participant` | `(label) → id` | 시퀀스 참여자 추가 |
| `connect` | `(from, to, label?)` | 엣지(플로우) / 메시지(시퀀스) 연결 |
| `relabel` | `(ref, newLabel)` | 노드/참여자/엣지 라벨 변경 |
| `remove` | `(ref)` | 요소 삭제 |
| `set_direction` | `(dir: "TD"\|"LR")` | 플로우차트 방향 |
| `switch_type` | `(target)` | 현재 IR을 대상 타입으로 의미 재구성(데모 하이라이트) |
| `export` | `(format: "mermaid"\|"png")` | 내보내기 |
| `clear` | `()` | 전체 삭제 — **사람 확인 후 실행** |

### 7.2 에이전트 운영 원칙

- **컨텍스트:** 매 턴 현재 IR(요약 포함)과 최근 대화를 모델에 전달 → "방금 그거" 같은 지시를 정확히 해석.
- **스트리밍:** 도구 호출 → IR 변경 → 렌더를 토큰/이벤트 단위로 흘려보내 지연을 체감시키지 않음.
- **검증:** 도구 적용 후 IR을 스키마 검증, 깨진 참조(존재하지 않는 id)면 에이전트에 피드백해 자가 수정.
- **결정적 출력:** 모델은 자유 텍스트로 Mermaid를 만들지 않는다 — 오직 도구로 IR을 바꾼다(환각·문법오류 차단).

## 8. IR(중간 표현) 스키마

```ts
type FlowchartIR = {
  type: "flowchart";
  direction: "TD" | "LR";
  title?: string;
  nodes: { id: string; label: string; shape?: "rect" | "round" | "diamond" }[];
  edges: { id: string; from: string; to: string; label?: string }[];
};

type SequenceIR = {
  type: "sequence";
  title?: string;
  participants: { id: string; label: string }[];
  messages: { id: string; from: string; to: string; label: string;
              kind?: "sync" | "async" | "return" }[];
};

type DiagramIR = FlowchartIR | SequenceIR;
```

- **직렬화기:** `serialize(ir): string` — IR을 유효한 Mermaid 텍스트로 변환하는 순수 함수(타입별 구현).
- **라벨 리졸버:** `resolve(ref, ir): id` — 자연어 참조를 IR 요소 id로 매핑(정확 일치 → 부분 일치 → 최근 언급 순).

## 10. Responsible AI · 보안 (심사 6%)

- **파괴적 작업 확인:** `clear`(전체 삭제) 등은 실행 전 사용자 확인 단계.
- **출력 투명성:** 에이전트가 "무엇을 왜 바꿨는지"를 도구 호출 로그로 노출 → 사용자가 주도권 유지.
- **환각 완화:** 모델은 IR만 변경하고 직렬화는 결정적 → 잘못된 문법/유령 요소 차단, 깨진 참조는 자가 수정 루프.
- **시크릿:** 모든 키는 Key Vault/환경변수. 프런트로 노출 금지, 리포 커밋 금지.
- **프롬프트 인젝션 인지:** 사용자 발화는 "데이터"로 취급, 시스템 프롬프트 권한을 넘기지 않음.

| 16:00–16:30 | (키보드 재허용 구간) 배포 마무리·제출·**데모 리허설** |

> 우선순위 가드레일: 막히면 **시퀀스/타입 전환을 stretch로 내리고**, "플로우차트 생성+편집+내보내기+배포"의 end-to-end 완주를 사수한다. 미제출 = 자동 탈락.

### 12.4 4단계 고도화 — 음성 교정과 반응속도 최적화

이 단계는 코어 제출 범위를 넘어서는 후속 고도화로 두되, 데모 직전 여유가 있으면 붙일 수 있는 개선으로 설계한다.

- 목표: 음성 입력의 오타, 한/영 오인식, 도메인 용어 오류를 빠르게 교정해 사용자가 체감하는 반응속도를 높인다.
- 방식: 규칙 기반 정규화로 먼저 치환하고, 모호한 경우에만 Copilot SDK 또는 Azure OpenAI 기반 교정 제안을 요청한다.
- UX: 교정 전/후를 한 줄로 비교해 보여주고, 신뢰도가 높을 때만 자동 적용한다.
- 안정성: 교정이 늦어지면 원문 또는 규칙 교정 결과를 바로 실행하고, 위험 명령은 기존 확인 플로우를 유지한다.
- 우선순위: 코어 기능을 다 끝낸 뒤에 붙인다. 느려지면 빼도 되는 stretch로 관리한다.

## 14. 성공 기준 (Definition of Done)

- Azure 배포 URL에서 **음성으로** 플로우차트 생성 → 증분 편집 → 내보내기가 end-to-end 동작.
- 시퀀스 타입 및 타입 전환이 데모에서 무중단 시연(코어 미달 시 stretch로 처리하되 코어는 필수).
